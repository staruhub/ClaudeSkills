# 详细代码审查检查清单

当需要进行更深入的代码审查时，参考此清单。

## 一、功能正确性

### 逻辑检查
- [ ] 代码逻辑是否与需求一致？
- [ ] 条件判断是否正确（注意边界值）？
- [ ] 循环是否有正确的终止条件？
- [ ] 递归是否有正确的基准情况？
- [ ] 状态变更是否按预期顺序执行？

### 边界条件
- [ ] 空输入如何处理？
- [ ] 最大/最小值如何处理？
- [ ] 负数、零值如何处理？
- [ ] 超长字符串如何处理？
- [ ] 并发请求如何处理？

## 二、安全性

### 输入验证
- [ ] 所有用户输入是否验证？
- [ ] 是否防止SQL注入？
- [ ] 是否防止XSS攻击？
- [ ] 是否防止命令注入？
- [ ] 文件上传是否限制类型和大小？

### 认证授权
- [ ] 敏感操作是否需要认证？
- [ ] 权限检查是否在服务端执行？
- [ ] 会话管理是否安全？
- [ ] 密码是否正确加密存储？
- [ ] Token是否有过期机制？

### 数据保护
- [ ] 敏感数据是否加密传输（HTTPS）？
- [ ] 敏感数据是否加密存储？
- [ ] 日志是否脱敏敏感信息？
- [ ] 错误信息是否泄露内部细节？
- [ ] API密钥是否硬编码？

## 三、性能

### 算法效率
- [ ] 时间复杂度是否合理？
- [ ] 是否有不必要的嵌套循环？
- [ ] 是否有重复计算可缓存？
- [ ] 大数据集是否分页处理？
- [ ] 是否有N+1查询问题？

### 资源管理
- [ ] 数据库连接是否复用？
- [ ] HTTP连接是否复用？
- [ ] 大对象是否及时释放？
- [ ] 是否有内存泄漏风险？
- [ ] 是否有文件句柄泄漏？

### 并发优化
- [ ] 是否利用并发提升效率？
- [ ] 锁的粒度是否合适？
- [ ] 是否有死锁风险？
- [ ] 线程池大小是否合理？
- [ ] 异步操作是否正确处理？

## 四、可读性与可维护性

### 命名规范
- [ ] 变量名是否清晰表达含义？
- [ ] 函数名是否动词开头？
- [ ] 类名是否名词且首字母大写？
- [ ] 常量是否全大写下划线分隔？
- [ ] 是否避免缩写和单字母变量？

### 代码结构
- [ ] 函数是否只做一件事？
- [ ] 函数长度是否合理（<50行）？
- [ ] 嵌套层级是否过深（>3层）？
- [ ] 是否有重复代码可抽取？
- [ ] 模块划分是否合理？

### 注释文档
- [ ] 复杂算法是否有注释？
- [ ] 公共API是否有文档？
- [ ] 特殊处理是否说明原因？
- [ ] 注释是否与代码一致？
- [ ] TODO是否有负责人和时间？

## 五、错误处理

### 异常处理
- [ ] 是否捕获可能的异常？
- [ ] 异常是否记录足够信息？
- [ ] 是否避免空的catch块？
- [ ] 是否区分可恢复和不可恢复错误？
- [ ] finally块是否正确释放资源？

### 失败处理
- [ ] 网络失败是否有重试机制？
- [ ] 超时是否设置合理？
- [ ] 服务降级方案是否就绪？
- [ ] 是否有断路器保护？
- [ ] 错误是否友好返回给用户？

## 六、测试覆盖

### 单元测试
- [ ] 核心逻辑是否有单元测试？
- [ ] 边界条件是否测试覆盖？
- [ ] 异常路径是否测试覆盖？
- [ ] 测试用例是否独立可重复？
- [ ] Mock是否合理使用？

### 测试质量
- [ ] 测试是否验证正确的行为？
- [ ] 断言是否清晰明确？
- [ ] 测试数据是否有代表性？
- [ ] 是否避免测试实现细节？
- [ ] 测试失败信息是否有帮助？

## 七、框架特定检查

### React/Vue前端
- [ ] 组件是否合理拆分？
- [ ] 状态是否最小化？
- [ ] 是否避免不必要的渲染？
- [ ] 副作用是否正确清理？
- [ ] Key是否稳定唯一？

### Flutter移动端
- [ ] Widget树是否合理？
- [ ] 是否使用const构造函数？
- [ ] State是否正确管理？
- [ ] 是否处理生命周期？
- [ ] 是否适配不同屏幕？

### Spring/Node后端
- [ ] 事务边界是否正确？
- [ ] 连接池是否配置？
- [ ] 是否有慢查询？
- [ ] 缓存策略是否合理？
- [ ] API版本是否管理？

## 使用说明

1. **日常审查**：使用SKILL.md中的快速检查清单
2. **深度审查**：结合本文档进行全面检查
3. **安全审计**：重点关注第二节安全性检查
4. **性能优化**：重点关注第三节性能检查
5. **代码重构**：重点关注第四节可读性检查
