# 场景识别与风格推荐规则

## Step 1：任务类型判定

| 信号 | 任务类型 |
|------|---------|
| 给领导/客户汇报进展、季度总结 | 汇报 |
| 融资、评委、"讲故事"、时间限制 | 路演 |
| 课件、教学、"给学员" | 培训 |
| 争取预算/立项/合作 | 提案 |
| 复盘、经验总结、post-mortem | 复盘 |
| 交付给甲方的分析报告 | 咨询报告 |
| 团队/公司/个人介绍 | 招聘介绍 |
| 发小红书/知乎/公众号的图 | 自媒体分享 |

多信号冲突或都不像时：直接问"这套 PPT 是给谁看、想达成什么"，不猜。

## Step 3：场景 × 风格推荐矩阵

推荐 3 个，必须跨 ≥2 个类别（防同质化）。

| 任务类型 | 首推 | 备选 |
|---------|------|------|
| 汇报 / 咨询报告 | 墨白咨询 | 英黄工作室、铂灰未来 |
| 数据/财务汇报 | 黑白账本 | 墨白咨询、铂灰未来 |
| 路演 / 提案 | 英黄工作室 | 极夜科技、糖盒彩框（青年向）|
| 培训 | 清风讲堂 | Notion 手绘、墨白咨询 |
| 学术答辩 / 研究汇报 | 学术蓝灰 | 墨白咨询、清风讲堂 |
| 复盘 / 招聘介绍 | 墨白咨询 | 糖盒彩框、清风讲堂 |
| 自媒体分享 / 知识图解 | 热帖卡片 | Notion 手绘、ChaoGeek 像素半调 |
| AI/科技产品相关（任意类型） | 极夜科技（暗场）/ 铂灰未来（明场） | ChaoGeek 像素半调、墨白咨询 |
| 品牌内容（ChaoGeek / ClawTime / WorkBuddy） | 对应 custom/ 风格（锁定，不推荐替代） | — |

数据密集或需频繁改字的场景 → 推荐标注"默认模式 B"的风格（黑白账本、学术蓝灰），并主动说明模式 C 不适用的原因。

## 上下文追问优先级

信息不全时按此顺序问（最多 5 项，已知的跳过）：
1. 给谁看（决定语气与密度）
2. 目标（说服/告知/教学 → 决定结构骨架）
3. 页数（决定 outline 粒度）
4. 内容清晰 vs 视觉冲击（决定风格类别倾向与模式 B/C）
5. 指定风格或品牌色（有 → 跳过推荐）

## 结构骨架速查（outline 生成用）

- 说服型（提案/路演）：问题 → 方案 → 证据 → 行动
- 告知型（汇报/复盘）：结论先行 → 分项展开 → 风险 → 下一步
- 教学型（培训）：目标 → 概念 → 示例 → 练习 → 小结
