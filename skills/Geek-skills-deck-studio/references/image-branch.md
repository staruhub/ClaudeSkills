# 模式 C：image 模型视觉分支

## 纪律（重复一遍，因为最容易违反）

1. outline 用户确认之前，不生成任何图
2. 每页顺序：page brief（schema 填完）→ image prompt（按模板组装）→ 出图
3. image 模型不决定内容，只渲染已定义的页

## 适用 / 不适用

**适用**：路演视觉稿、社媒传播组图、信息图、需要"设计感成品"而非可编辑文件的场合。
**不适用**：需要频繁改字的工作文档（改一个数字要重画整页）、表格密集的财务/数据汇报（image 模型渲染表格不可靠）、>20 页的长 deck（成本与一致性都不划算，建议模式 B）。判定后主动告知用户代价，让用户选。

## 引擎适配（引擎无关设计）

page brief → image prompt 的组装逻辑是本 skill 的资产，与引擎解耦。最后一步按用户环境可用引擎适配：

| 引擎 | 适配要点 |
|------|---------|
| gpt-image 系 | 中文文字渲染较稳；prompt 用自然段描述；比例参数 16:9 |
| seedream（火山） | 走 `seedream-imagegen` skill 调用；2K 起；参考图控风格一致性 |
| skywork-design | 走 skywork skill；适合单页精修与改图迭代 |

引擎不可用时降级：输出完整的逐页 prompt 文本交付用户自行生成，流程照走。

## 逐页 prompt 必含要素（模板见 templates/image-prompt-template.md）

页面用途 / 风格名+style brief 关键参数 / 构图（版式代号展开为空间描述）/ 标题文案（原文）/ 正文文案（≤30 字或留位）/ 图表与图像要求 / 留白与层级要求 / 中英文排版要求 / 禁止项。

## 中文排版禁止项（每页 prompt 末尾原样附加）

```
禁止：文字重叠、文字压在复杂图像上、伪汉字乱码、
逐字竖排英文、超过两种字体、页面四边贴字无留白、
装饰元素盖过标题、每页超过 5 个信息块。
```

正文超过 30 字的页：改用"留位策略"——让 image 模型只画版式与视觉、留出文本区域，文字由后期（PPT/图片编辑器）叠加。这是中文长文本唯一可靠的做法。

## 整套一致性

- 同一 style brief 注入每页；页与页之间只变内容参数
- 生成顺序：先封面（定调）→ 抽 1 页正文页验证风格 → 用户认可后批量其余页
- 每 5 页抽查一次与封面的风格一致性；漂移则复用封面 prompt 的风格段重生成

## 避免"好看但不好讲"

出图前检查 page brief 的 key_message 与 speaker_notes 是否已填——图是给观众看的，notes 是给讲者用的，两者都交付。
