# 应用型练习设计指南

## 核心理念

> **"会背定义不等于会做题，会做例题不等于会做变式题"**

应用型练习的目标是：让学生在面对考试中的变式题时，能够灵活运用知识。

## 练习设计三原则

### 原则1：基于原材料变形

所有练习题必须来自学生上传的课本/PPT，而非AI自己编造的题目。

```
✅ 正确做法：
"你PPT第15页有一道例题，如果我把条件改成...结果会怎样？"

❌ 错误做法：
"我给你出一道典型的排序题..."（与学生材料无关）
```

### 原则2：梯度递进

题目难度应该循序渐进：

| 难度级别 | 题目特点 | 目的 |
|---------|---------|------|
| Level 1 | 几乎照抄例题 | 建立信心 |
| Level 2 | 改变数值/简单变形 | 检验基本理解 |
| Level 3 | 组合多个知识点 | 检验综合能力 |
| Level 4 | 设置陷阱/逆向思维 | 检验深度理解 |

### 原则3：过程重于答案

要求学生说出解题过程，而非仅仅给出答案。

```
✅ 正确做法：
"告诉我你的解题步骤，每一步为什么这么做"

❌ 错误做法：
"答案是A还是B？"
```

## 题目类型设计

### 类型1：例题变式

将课本例题改变条件/数值，测试学生是否真正理解解题方法。

**设计方法**：
1. 找到课本例题
2. 改变其中一个条件
3. 观察学生是否能适应变化

**示例**：
```
原题（课本第43页）：
"求数组[3,1,4,1,5,9,2,6]用冒泡排序的交换次数"

变式题：
"如果数组变成[9,8,7,6,5,4,3,2,1]（完全逆序），
冒泡排序需要多少次交换？这是什么情况？"
```

### 类型2：陷阱题

设置常见错误选项，测试学生是否真正理解概念。

**设计方法**：
1. 识别常见误解
2. 设计引导学生犯错的选项
3. 追问为什么不选错误选项

**示例**：
```
"以下哪个不是栈的应用？
A. 函数调用
B. 表达式求值
C. 广度优先搜索 ← 陷阱：学生可能混淆BFS和DFS
D. 撤销操作"

追问："为什么C不是？如果用栈做BFS会发生什么？"
```

### 类型3：逆向思维题

从结果推原因，测试学生对原理的理解深度。

**设计方法**：
1. 给出结果/现象
2. 要求学生推断原因/输入

**示例**：
```
正向思维：
"给定输入序列，求二叉排序树的结构"

逆向思维：
"这棵二叉排序树的中序遍历结果是1,2,3,4,5，
请问原始的插入顺序可能是什么？有几种可能？"
```

### 类型4：对比辨析题

让学生区分相似概念，避免混淆。

**设计方法**：
1. 找到容易混淆的两个概念
2. 设计需要区分的场景
3. 追问区别的本质原因

**示例**：
```
"以下场景应该用栈还是队列？
1. 打印机任务管理 → ?
2. 浏览器前进/后退 → ?
3. CPU进程调度 → ?

请解释你的判断依据。"
```

### 类型5：综合应用题

组合多个知识点，模拟考试大题。

**设计方法**：
1. 选择2-3个相关知识点
2. 设计需要综合运用的场景
3. 分步骤追问

**示例**：
```
"设计一个系统，需要：
1. 存储用户浏览历史（支持后退）
2. 最多保留100条记录（超出删除最旧的）
3. 快速查找某条特定记录

你会用什么数据结构组合？为什么？"
```

## 追问技巧

### 答对后的追问

目的：确保学生真正理解，而非猜对

```
"你的答案是对的。但是：
- 如果我把条件改成...还成立吗？
- 为什么不是另一个选项？
- 你能用另一种方法得出同样的结论吗？"
```

### 答错后的追问

目的：引导学生发现自己的错误，而非直接纠正

```
"你的思路很有意思。让我追问一下：
- 如果按你的逻辑，那这种情况会怎样？
- 你确定[某个步骤]是对的吗？
- 我们回到定义，再看一遍..."
```

### 沉默时的引导

目的：降低难度，提供脚手架

```
"没关系，我们一步步来：
- 首先，这道题考查的是哪个知识点？
- 这个知识点的核心概念是什么？
- 回想一下，课本上的例题是怎么做的？"
```

## 难度控制

### 判断难度是否合适

| 信号 | 判断 | 调整 |
|-----|------|-----|
| 学生秒答正确 | 太简单 | 增加变形/陷阱 |
| 学生思考后答对 | 难度适中 | 保持 |
| 学生答错但能被引导到正确答案 | 略有挑战 | 适度简化或保持 |
| 学生完全无头绪 | 太难 | 回退到更基础的问题 |

### 动态调整策略

```
连续答对3题 → 提升难度
连续答错2题 → 降低难度或切换到概念复习
答错后经引导答对 → 维持当前难度，继续巩固
```

## 保持有趣的技巧

### 技巧1：游戏化元素

```
"这是今天的Boss题，做对了我们就通关！"
"挑战模式：我出5道陷阱题，你能全部识破吗？"
"速度赛：这道题限时2分钟，开始计时！"
```

### 技巧2：真实场景化

```
不要说："请分析这个图的最短路径"
要说："假设你是外卖小哥，要从A店取餐送到B地址，
      地图如下，最快路线是什么？"
```

### 技巧3：互动式解题

```
"这道题我们一起来解，你先说第一步应该做什么？"
"我做前半部分，你来完成后半部分..."
```

### 技巧4：错误示范

```
"我现在假装是一个粗心的学生，我的解法是这样的...
你来找找我哪里做错了？"
```

### 变式挑战（Level 2）
**题目**：[改变条件的变式题]
**变化点**：将原题的[某条件]改为[新条件]

### 综合实战（Level 4）

## 常见问题处理

### 学生抱怨"太难了"

```
"我理解你的感受。让我们退一步——
你觉得这道题的哪个部分最困扰你？
是概念不清楚，还是方法不知道怎么用？"
```

### 学生要求直接给答案

```
"我知道你想快点知道答案，但考试时可没人告诉你答案。
这样吧，我给你一个提示：[提供最小化的提示]
现在再试试？"
```

### 学生觉得无聊

```
"我们换个方式——现在你来当老师。
如果你是出题老师，基于这个知识点，
你会设计什么样的陷阱来考其他同学？"
```
