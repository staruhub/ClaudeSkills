# 苏格拉底教学法指南

## 核心理念

苏格拉底式教学法的本质是：**通过精心设计的提问，引导学生自己发现答案，而非直接灌输知识**。

在备考场景中，这意味着：
- 学生记住的不只是答案，而是**得出答案的思考过程**
- 遇到变式题时能举一反三
- 知识真正内化，而非短期记忆

## 提问层次金字塔

从易到难，分层提问：

### Level 1：事实性问题（热身）
- "根据PPT第X页，这个概念的定义是什么？"
- "课本上提到了哪几种类型？"
- **目的**：检验学生是否看过材料

### Level 2：理解性问题（核心）
- "为什么会这样设计？"
- "如果去掉这个条件，结果会怎样？"
- "这个和之前学的XX有什么联系？"
- **目的**：检验学生是否真正理解

### Level 3：应用性问题（实战）
- "用这个原理解释一下这道题..."
- "如果我把条件改成...结果会怎样？"
- "这个概念在什么情况下会失效？"
- **目的**：检验学生能否运用知识

### Level 4：分析评价问题（进阶）
- "这两种方法各有什么优缺点？"
- "你觉得作者为什么要用这种方式表达？"
- "如果你是出题老师，你会怎么考这个知识点？"
- **目的**：培养深度思考能力

## 实战提问技巧

### 技巧1：反问追击

学生给出答案后，不要直接说对错，而是追问：

| 学生答对时 | 学生答错时 |
|-----------|-----------|
| "你是怎么想到的？" | "你能再解释一下你的思路吗？" |
| "为什么不是另一个选项？" | "如果按你的逻辑，那这种情况怎么解释？" |
| "还有别的方法吗？" | "我们回到定义，再看一遍..." |

### 技巧2：对比启发

当学生混淆相似概念时：

```
❌ 直接告诉区别
"栈是LIFO，队列是FIFO，它们的区别是..."

✅ 对比提问
"想象两个场景：
1. 银行取号排队
2. 弹匣装子弹
哪个是栈？哪个是队列？为什么？"
```

### 技巧3：极端假设

通过极端情况帮助理解本质：

```
"如果数组只有一个元素，二分查找会怎样？"
"如果网络完全不丢包，TCP还需要确认机制吗？"
"如果内存无限大，我们还需要虚拟内存吗？"
```

### 技巧4：错误引导

故意给出错误思路，让学生纠正：

```
"我觉得这道题应该用XX方法解，你觉得可行吗？"
（实际上这个方法不适用）

"有人说这个结论在所有情况下都成立，你同意吗？"
（实际上有反例）
```

### 技巧5：角色互换

让学生当老师：

```
"假设你要给一个完全不懂的同学解释这个概念，你会怎么说？"
"如果你是出题老师，这个知识点你会怎么设计陷阱？"
```

## 对话节奏控制

### 避免问题轰炸

❌ 错误示范：
> "好，那你告诉我什么是栈？栈有什么特点？栈可以用来做什么？栈和队列有什么区别？"

✅ 正确示范：
> "好，让我们从栈开始。你能想到生活中什么东西是'后进先出'的吗？"
> （等待学生回答）
> "很好！那么..."

### 给予思考时间

- 提问后等待3-5秒
- 学生沉默时说"不着急，慢慢想"
- 复杂问题可以说"你可以先说说你现在的想法，哪怕不完整"

### 适时给予提示

当学生明显卡住时，给予阶梯式提示：

1. **提示1**（最小）："回想一下PPT第X页的内容..."
2. **提示2**（中等）："我们之前说过，XX和YY有一个共同点..."
3. **提示3**（较多）："换个角度想，如果条件是..."

## 处理不同类型的学生

### 自信但错误的学生
- 不要直接否定
- 用反问揭示矛盾："按你的逻辑，那这种情况怎么解释？"

### 沉默不愿回答的学生
- 降低问题难度
- 给予选择题式提问："你觉得是A还是B？"
- 鼓励任何尝试："说说你的第一反应"

### 急于求答案的学生
- 明确规则："我们的目标是你自己能想出来，这样考试才能用上"
- 承诺回报："你自己想出来的东西，比我告诉你的记得牢10倍"

### 基础很差的学生
- 从最基础的事实性问题开始
- 不要急于进入应用层
- 多用类比和生活化例子

## 备考场景专用话术

### 开场
- "我们今天复习XX章，你对这章最没把握的是哪个部分？"
- "在开始之前，你能用自己的话概括一下这章讲了什么吗？"

### 过程中
- "你这个理解对考试够用了，不过我想追问一下..."
- "如果考试题这么出，你能做对吗？"
- "这个地方老师可能会设陷阱，你知道陷阱在哪吗？"

### 总结时
- "今天我们讨论了三个重点，你能回顾一下吗？"
- "你觉得现在对这个知识点有多少把握？0-10分？"
- "还有什么不清楚的地方吗？"

## 示例对话

### 场景：学生复习"操作系统-进程调度"

> **Claude**：好的，我看了你上传的PPT。在进程调度这一章，你觉得自己最薄弱的是哪个部分？
>
> **学生**：时间片轮转和优先级调度我老是搞混。
>
> **Claude**：理解。那让我问你一个场景：
>
> 假设你是银行经理，银行里有三个窗口，但有10个客户在等待。每个客户的业务需要不同的时间。你会怎么安排？
>
> **学生**：按先来后到？
>
> **Claude**：这是一种方式，叫FCFS。但现在情况变了——假设其中一个客户是银行VIP，只需要1分钟办完；另一个普通客户要办30分钟。你还会坚持先来后到吗？
>
> **学生**：那可能让VIP先办，效率更高。
>
> **Claude**：对！这就涉及到优先级的概念了。那我再问你——如果VIP客户的业务也要30分钟呢？普通客户会不会不满？
>
> **学生**：会啊...他们会一直等。
>
> **Claude**：没错，这就是优先级调度可能造成的"饥饿"问题。那根据你PPT第23页，时间片轮转是怎么解决这个问题的？
